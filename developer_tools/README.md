# GitHub Shell 开发者工具

这是 GitHub Shell 项目的开发者专用文件夹，包含各种开发者工具和快捷方式，方便开发者使用。

## 目录结构

```
developer_tools/
├── switch_to_developer_mode.py  # 快速切换到开发者模式
├── run_tests.py                 # 运行测试套件
├── build_package.py             # 构建包
└── README.md                    # 本说明文档
```

## 开发者模式介绍

GitHub Shell 有两种运行模式：

### 1. 用户模式
- 适合普通用户使用
- 功能受限，无法修改核心配置
- 无法执行危险操作

### 2. 开发者模式
- 适合开发者使用
- 可以修改核心配置
- 可以执行所有功能，包括危险操作
- 需要密码验证

## 工具说明

### 1. 切换到开发者模式

**脚本名称**: `switch_to_developer_mode.py`

**功能**: 快速切换到开发者模式，无需手动输入命令

**使用方法**: 
```bash
python developer_tools/switch_to_developer_mode.py
```

**执行流程**:
1. 检查开发者模式是否被锁定
2. 提示输入开发者密码
3. 验证密码并切换到开发者模式

### 2. 运行测试套件

**脚本名称**: `run_tests.py`

**功能**: 运行项目的测试套件，验证代码质量

**使用方法**: 
```bash
python developer_tools/run_tests.py
```

**执行流程**:
1. 运行所有测试用例
2. 显示测试结果
3. 如果有测试失败，返回错误码

### 3. 构建包

**脚本名称**: `build_package.py`

**功能**: 构建项目包，生成可发布的文件

**使用方法**: 
```bash
python developer_tools/build_package.py
```

**执行流程**:
1. 安装构建依赖
2. 构建包
3. 显示构建结果和产物

## 开发者模式命令

切换到开发者模式后，您可以使用以下特殊命令：

| 命令 | 功能 |
|------|------|
| `mode developer` | 切换到开发者模式（需要密码） |
| `mode user` | 切换回用户模式 |
| `test <command>` | 测试指定命令 |
| `reset-config` | 重置配置 |
| `developer-password set` | 设置开发者密码 |
| `developer-password clear` | 清除开发者密码 |

## 常见问题

### Q: 忘记开发者密码怎么办？
A: 请联系项目管理员重置密码。

### Q: 开发者模式被锁定怎么办？
A: 请联系项目管理员解锁。

### Q: 运行脚本时提示缺少依赖？
A: 请确保已安装所有依赖：
```bash
pip install -e .
pip install pytest build twine
```

### Q: 如何退出开发者模式？
A: 执行命令：`mode user`
